# План реализации: Автоматизация GitHub Issues

## Обзор

Этот план разбивает систему автоматизации GitHub Issues на небольшие, проверяемые шаги. Реализация идет инкрементально на TypeScript: сначала базовая инфраструктура и модели, затем AI-классификация, поиск дублей, и в конце воркфлоу жизненного цикла (закрытие дублей и устаревших тикетов).

## Примечания

- Задачи с `*` опциональны и могут быть пропущены для быстрого MVP
- Каждая задача ссылается на требования для трассируемости
- Контрольные точки обеспечивают поэтапную валидацию
- Property-тесты проверяют универсальные свойства корректности
- Unit-тесты проверяют конкретные примеры и крайние случаи
- Весь TypeScript-код должен следовать best practices и использовать strict mode
- Используй аннотации типов повсеместно для качества кода
- AWS credentials должны храниться в GitHub Secrets, никогда в коде


## Объяснение на пальцах
- Объясняй сложные технические вещи простым языком, как новичку.
- Выстраивай логику пошагово: что это, зачем нужно, как работает, какой результат.
- Где уместно, добавляй короткий практический пример.
- Цель ответа: новичок уже сегодня может написать качественный код и при этом понимает логику решения.
