# Design Format

## Суть

`Design.md` отвечает на вопрос:

- как именно это делать
- из каких частей состоит решение
- как проверять и выкатывать

## Обязательная структура

1. `# Design Document`
2. `## Обзор`
3. `## Архитектура`
4. `## Компоненты и интерфейсы`
5. `## Модели данных`
6. `## Свойства корректности`
7. `## Обработка ошибок`
8. `## Стратегия тестирования`
9. `## План выката`
10. `## Объяснение на пальцах`

## Правила

1. Пиши на русском.
2. Делай дизайн привязанным к требованиям.
3. Отдельно описывай rollout и проверку.
4. Если есть спорные места, объясняй, почему выбран именно этот путь.
