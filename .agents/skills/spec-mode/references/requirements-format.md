# Requirements Format

## Суть

`Requirements.md` отвечает на вопрос:

- что хотим получить
- зачем это нужно
- как понять, что задача реально сделана

## Обязательная структура

1. `# Requirements Document`
2. `## Введение`
3. `## Глоссарий`
4. `## Требования`
5. Для каждого требования:
   - заголовок `### Требование N: ...`
   - пользовательская история
   - `#### Критерии приёмки`
6. `## Свойства корректности`
7. `## Объяснение на пальцах`

## Правила

1. Пиши на русском.
2. Формулируй критерии через:
   - `КОГДА ...`
   - `ТО система ДОЛЖНА ...`
3. Делай требования проверяемыми.
4. Не смешивай требования с реализацией.
5. В конце обязательно простой блок для новичка.
