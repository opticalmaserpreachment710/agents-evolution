# ELI5 — Explain Like I'm 5

Максимально простое объяснение. Без жаргона. Без предположений о знаниях.

## Когда использовать

- «Объясни как ребёнку» / «eli5» / «на пальцах» / «по-простому»
- Пользователь явно говорит что не разбирается в теме
- Нужно быстро донести суть без деталей

## Правила ELI5

### Уровень языка

- Чтение: 6-7 класс
- Предложения: короткие. 1 мысль = 1 предложение
- Нет терминов без мгновенного `термин = (пояснение в скобках)`
- Нет абстрактных концепций без «это как...»

### Структура

```
1. Суть: [1 предложение. Что делает/зачем нужно]
2. Это как: [бытовая аналогия]
3. Вот что происходит: [2-3 простых шага, без кода]
4. Пример из жизни: [конкретная ситуация]
```

## Шаблоны промптов

### Шаблон 1: Объясни код

```
Объясни этот код как будто мне 10 лет.
- Не используй профессиональные термины. Если без термина нельзя — поясни его сразу простыми словами.
- Используй аналогию из реальной жизни.
- Покажи на конкретном примере что происходит.

[вставь код]
```

### Шаблон 2: Объясни концепцию

```
Что такое [концепция]? Объясни так, чтобы понял человек без технического образования.
- Начни с 1 предложения.
- Дай аналогию из быта (магазин, почта, стройка, готовка, транспорт).
- Приведи 1 конкретный пример из жизни.
- НЕ упоминай реализации, библиотеки, паттерны.
```

### Шаблон 3: Переведи с технического на человеческий

```
Перепиши этот текст так, чтобы понял непрофессионал.
Сохрани техническую точность, но убери жаргон.
Каждый термин замени на описание что он делает.

[вставь технический текст]
```

### Шаблон 4: Почему это сломалось

```
Объясни простыми словами почему [вещь] не работает.
- Не говори что «очевидно».
- Не предполагаешь что я знаю.
- Покажи шаг за шагом что пошло не так.
- Используй аналогию («это как если бы...»).

[вставь ошибку или описание проблемы]
```

## Примеры хороших ELI5

### useMemo

**Плохо:**
> useMemo мемоизирует результат вычисления и пересчитывает только при изменении зависимостей.

**Хорошо:**
> Представь калькулятор. Ты спросил «сколько 2+2?» — он ответил 4. Если спросить ещё раз — он посчитает заново, хотя ответ тот же. `useMemo` — это стикер на калькуляторе: «ответ 4, не считай снова». Но если числа поменялись (2+3) — он пересчитает.

### Promise

**Плохо:**
> Promise — объект представляющий завершение асинхронной операции.

**Хорошо:**
> Ты заказал пиццу по телефону. Оператор сказал: «будет через 30 минут». У тебя есть обещание (promise) что пицца приедет. Пока она едет — ты делаешь другие дела. Когда приехала — ты её забираешь. Promise — это обещание что результат будет, просто не прямо сейчас.

### Rate Limiter

**Плохо:**
> Rate limiter ограничивает количество запросов за единицу времени для предотвращения перегрузки.

**Хорошо:**
> Представь турникет в метро. Ты можешь пройти — но только один за раз. Если бежать толпой — турникет заблокируется. Rate limiter — это турникет для запросов: пропускает по N штук в минуту, остальные — «подожди».

## Чего избегать

- ❌ «По сути...» — и дальше сложный текст
- ❌ Ссылки на MDN/docs вместо объяснения
- ❌ «Это тривиально» — для просящего не тривиально
- ❌ Диаграммы без текстового пояснения
- ❌ 5 абзацев когда хватает 3 предложений
